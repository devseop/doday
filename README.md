# Do-Day

## Project Summary

- 해야 하는 일들이 하루에 끝낼 수 없는 것들도 있는데 기존의 `To-do List`는 당일에만 완료되어야 하는걸까?
- 완료되어야 하는 시점을 정해놓고 유연하게 처리해낼 수 있는 `To-do List`가 있으면 어떨까?

**오늘 하루가 기준이 아닌 D-Day를 기준으로 `To-Do`를 계획하고 실행해보자**는 생각으로 시작한 토이 프로젝트입니다.

## Tech Stack

`React` `TypeScript` `Jotai` `Tailwind CSS`

## Commit Convention

- `feat`: 새로운 기능의 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드의 변경이 없는 포맷팅
- `refactor`: 코드 스팩토링
- `test`: 테스트 코드, 리팩토링 테스트 코드 추가
- `chore`: 빌드 업무 수정, 패키지 매니저 수정
- `set`: 프로젝트 셋팅
